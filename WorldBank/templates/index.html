TutorCruncher Test

Dev test for potential developers based on Django.

This is the first template you need to edit to build the views.

You are required to prepare a web application that will allow users to view world bank data on education.

The user should be able to view lists of metrics, filter these metrics (eg. by country or type), sort the data in ascending or descending order and view a particular Metric details.

You may use Django's admin interface during development, but your final result should not use Django admin interface.

You might like to think about the following things while building the site:

  • Easy of Use: the site should be as intuitive and easy to use as possible.
  • Aesthetics: the site should be modern and professional looking.
  • Canonical Development Approach: you should use the standard approach for building views and implementing functionality to make it as easy as possible for someone else to take of the project and add features.
  • Minimising Queries: to keep the site fast and avoid excessive server requirements if the site was used a lot.

The basic process you will need to follow is:

  1. Take a look at DevTest/load_data.py to work out how the data looks.
  2. Build the models in DevTest/WorldBank/models.py to represent that data.
  3. Import the data by modifying DevTest/load_data.py.
  4. Build the views to display the data.

To get started

It's assumed here you are working on Linux or OSX and have python3.9 installed.

Setup a virtual environment and enable it (you may need to install virtualenv first):


      virtualenv env
      source env/bin/activate

install the required packages:


      pip install -r requirements.txt

You can then run the dev-server with:


      python manage.py runserver

Once you have build your models you can build the database with:


      python manage.py migrate
